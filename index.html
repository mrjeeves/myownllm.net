Self-host the JSON, set it, forget it. MyOwnLLM gives you an OpenAI-compatible server, a desktop GUI, and a static manifest format that picks the right model for whatever hardware it lands on — from a 2 GB Pi 4 to a 4090.

What you actually get

One stable URL, swappable model

Point Cursor, Continue, Aider, Cline, Zed, Open WebUI, LibreChat, opencode — anything OpenAI-shaped — at http://127.0.0.1:1473/v1 and forget about it. The virtual model IDs myownllm-text, myownllm-code, myownllm-vision, myownllm-transcribe resolve at request time to the best tag your machine can run.

Manifest-driven, not click-driven

A static JSON file decides which model runs where. Host your own, point a team at it, or compose merged catalogs with JSON imports. No coordination server, no auth flow, no dashboard — just a URL.

Hardware-tier resolution

The same manifest hands a 2 GB Pi a 1B model and a 4090 a 70B model. The hardware detector reads /proc/device-tree/model, nvidia-smi, rocm-smi, system_profiler — and walks tiers top to bottom.

Desktop GUI with live transcription

Tauri + Svelte 5. Two singleton slots — one chat model, one transcription. Conversation sidebar with folders, drag-to-organise, rename in place. Whisper.cpp runs offline; the mic-pause keeps draining the backlog. Talking Points continuously summarises the transcript into a bullet list.

Self-updating, crash-resilient

Checks GitHub on launch, stages quietly, applies on next start. Transcription chunks land on disk before whisper sees them, so a force-quit can be drained on the next launch. No network, no problem — the last good manifest is cached and kept.

Scriptable end to end

Every CLI subcommand returns parseable text or --json. myownllm status shows provider, hardware tier, ollama state, update status — one screen. SSE progress on preload. Pipe it, automate it, alarm on it.

An API your clients already speak

MyOwnLLM serves three wire formats from the same binary. Point any client at the matching port — no shim, no adapter, no proxy.

OpenAI http://127.0.0.1:1473/v1
  • POST /v1/chat/completions — streaming, tools, vision, JSON mode
  • POST /v1/completions
  • POST /v1/embeddings
  • GET  /v1/models
Ollama http://127.0.0.1:11434
  • POST /api/chat
  • POST /api/generate
  • GET  /api/tags
Anthropic /v1/messages
  • POST /v1/messages — SSE deltas, content blocks
  • Works with anything that speaks Claude's wire format

Drop-in curl

curl http://127.0.0.1:1473/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer myownllm" \
  -d '{
    "model": "myownllm-text",
    "messages": [{"role": "user", "content": "hi"}],
    "stream": true
  }'

Runs on a Raspberry Pi

Native linux-aarch64 builds. The detector reads the board name and walks a CPU-friendly tier ladder — a 2 GB Pi 4 lands on llama3.2:1b, a 16 GB Pi 5 reaches gemma4:e4b. Gemma 4's edge variants are agentic, multimodal, Apache-2.0, and run completely offline.

BoardDefault text model
Pi 4 / Pi 5 — 2 GBllama3.2:1b
Pi 4 / Pi 5 — 4 GBllama3.2:3b
Pi 4 / Pi 5 — 8 GBgemma4:e2b
Pi 5 — 16 GBgemma4:e4b

CLI

Every subcommand has a --json twin. Wire it into whatever you already have. 

$ myownllm                 # GUI
$ myownllm serve           # API server (127.0.0.1:1473)
$ myownllm run             # terminal chat
$ myownllm status          # provider, hardware, daemon, update
$ myownllm models          # what's pulled, what could be
$ myownllm families        # list / switch family
$ myownllm providers       # list / switch provider
$ myownllm update          # check / apply / configure self-update
